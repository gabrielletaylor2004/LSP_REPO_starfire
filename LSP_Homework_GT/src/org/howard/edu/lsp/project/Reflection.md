This project involved designing and deploying a Python-based Event Management System using Google Cloud Compute Engine. The goal was to demonstrate foundational cloud computing concepts through a working application.This was my first time working with cloud computing, and I initially found the process unfamiliar and challenging. I am relatively new to computer science, as I decided to pursue it as a minor after enjoying it in high school. Because of this, many of the concepts introduced in this course, including cloud infrastructure and deployment, were completely new to me.

Despite this, I was able to build a fully functional system using an object-oriented architecture, separating concerns across entities, repositories, services, and managers. This structure improved modularity and made the system easier to maintain and extend. One of the key cloud concepts demonstrated in this project was Infrastructure as a Service (IaaS). By using a virtual machine on Google Cloud, I was able to deploy and run my application in a remote environment without modifying the core logic. This helped me understand how cloud platforms allow applications to run independently of local machines. A major challenge was learning how to navigate the Google Cloud interface and properly configure the virtual machine. Additionally, transferring and updating code required troubleshooting, particularly when dealing with SSH connections, file uploads, and system behavior in a remote environment.

However, through completing this project, I learned a great deal by actively working through these challenges. It gave me hands-on experience with cloud deployment, strengthened my understanding of system design, and increased my confidence in using unfamiliar technologies. If I were to extend this project, I would integrate a persistent database and convert the system into a web-based application to improve accessibility and scalability.
